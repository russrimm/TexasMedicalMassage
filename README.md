# Texas Medical Massage

A modern, mobile-first marketplace connecting Texas massage therapists with massage businesses that are hiring. Built with Next.js 15, TypeScript, Tailwind, Auth.js, Drizzle ORM, and Supabase (Postgres + PostGIS).

## Features

- Two-sided sign-up: **Therapist** and **Business** profiles
- GPS-aware radius search (PostGIS `ST_DWithin`) for therapists, businesses, and jobs
- Job posting + applications with cover note
- 1–5 star reviews with "Verified Engagement" badge for participants who actually communicated
- Direct in-app messaging
- Mapbox geocoding ("use my location" + address autocomplete)
- Light/dark theme, mobile-first responsive UI

## Getting started

### 1. Install dependencies

```bash
npm install
```

### 2. Create a Supabase project

1. Create a new project at https://supabase.com (free tier is fine).
2. **Database → Extensions**: enable `postgis`, `pg_trgm`, and `uuid-ossp`. Or run `drizzle/bootstrap.sql` via the SQL editor or `psql`.
3. From **Settings → Database**, copy:
   - **Transaction pooler** URI (port 6543) → `DATABASE_URL`
   - **Session pooler** URI (port 5432)     → `DIRECT_URL`

### 3. Configure environment

Copy `.env.example` to `.env.local` and fill in:

```bash
cp .env.example .env.local
# generate AUTH_SECRET:
openssl rand -base64 32
```

Required for full functionality:

- `DATABASE_URL`, `DIRECT_URL`, `AUTH_SECRET`
- `MAPBOX_TOKEN` (server token, used by `/api/geocode`)
- `NEXT_PUBLIC_MAPBOX_TOKEN` (browser token for maps)

Optional:

- `AUTH_GOOGLE_ID` + `AUTH_GOOGLE_SECRET` — adds Google sign-in
- `RESEND_API_KEY` — transactional email
- `SUPABASE_SERVICE_ROLE_KEY` — for Storage uploads

### 4. Run migrations and seed dev data

```bash
npm run db:generate     # produces drizzle/migrations from src/server/db/schema.ts
npm run db:migrate      # applies migrations to your Supabase DB
npm run db:seed         # creates 6 therapists + 6 businesses + jobs across Texas cities
```

### 5. Start the dev server

```bash
npm run dev
```

Open http://localhost:3000.

**Seeded logins** (all password `password123`):

- Therapists: `therapist1@example.com` … `therapist6@example.com`
- Businesses: `business1@example.com` … `business6@example.com`

## Project structure

```
src/
  app/                 # App Router (pages + route handlers)
  components/
    ui/                # shadcn-style primitives
    site/              # header / footer / theme
    auth/              # sign-in / sign-up forms
    forms/             # profile + job + location-picker
    search/            # search panel + result cards
    reviews/           # review form
  server/
    auth/              # Auth.js config (edge-safe + node)
    db/                # Drizzle schema, client, migrate, seed
    queries/           # PostGIS search queries
    actions/           # Server actions: auth, profile, jobs, reviews, messages
  lib/                 # utils, validation, search-param parser
  middleware.ts        # auth + onboarding gates
drizzle/
  bootstrap.sql        # one-time extension setup
  migrations/          # generated by drizzle-kit
```

## Deploying to Vercel

1. Push to GitHub.
2. Import the repo into Vercel.
3. Add the same env vars (Project Settings → Environment Variables).
4. Vercel will run `next build` automatically.
5. Run `npm run db:migrate` from your local machine pointing at the production `DIRECT_URL`.
